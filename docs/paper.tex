\noindent\textit{Technical report (results-led). Every number below is produced by the committed harness on the
public mock + synthetic data, or on vanilla Odoo CE --- reproducible with no access to any private code.
Reference result tables live in \code{results/}.}

\begin{abstract}
Enterprises are wiring tool-calling LLM agents into ERP systems to answer business questions in natural
language. We identify an authorization failure these agents expose at the \textbf{data-result plane}: a tool
call that is perfectly valid at the \emph{control plane} (an allowed model, valid parameters) still returns
rows the user may not see, because the agent autonomously queries a \textbf{child model whose row-level record
rule is missing} while the parent's rule is present --- \emph{relational-traversal bypass}. This is a
confused-deputy / BOLA gap that warehouse-native governed-NL-analytics platforms (which inherit complete
governance from the warehouse) and action-plane authorization frameworks (which authorize the call, not the
rows) do not address for ERP. We contribute: (i) \textbf{ERP-AuthZBench}, an adversarial benchmark for
row-level authorization of ERP LLM agents; (ii) \textbf{PG-Agent}, a model-agnostic data-result-plane Policy
Enforcement Point (PEP) that is clean on every benchmark class; and (iii) \textbf{PCC-ERP}, a policy-closure
compiler that \emph{derives} the per-model row-level closures from the ORM relation graph + existing record
rules, emits them as enforceable policy, and runtime-verifies gap closure --- validated on the mock
end-to-end and on vanilla Odoo CE \code{sale}+\code{account}+\code{stock}. On the orthogonal reliability axis
(RQ6) we adopt a three-layer \textbf{integrity} stack (numeric verifier + governed metrics + execution-guided
self-consistency) that drives the silently-wrong-number rate to zero and catches
correct-arithmetic-with-the-wrong-formula --- framed as applied, not novelty. We additionally (iv)
regression-gate the residual-risk surface with a deterministic, LLM-free \textbf{red-team grammar} that
exhausts the ORM-pivot space (T4.5+); (v) \textbf{formalize} the bespoke POLICY as an instance of a general
\textbf{ABAC$\times$ReBAC} subject-context model whose compiler reproduces the guard's exact authorization
domain (RQ7); and (vi) extend the PEP to a \textbf{Doc-RAG retrieval plane} that delivers chunks only
re-rendered from row-authorized, clearance-masked sources (RQ8).

The contribution is scoped honestly to \textbf{applied security + benchmark + reference implementation} for an
under-served setting (ERP record-rule governance that is incomplete on child models), not to a novel
unification of authorization and integrity (prior art) nor to novel soundness on arbitrary policy domains.
\end{abstract}

\section{Problem}

\subsection{Control plane vs data-result plane}
\begin{itemize}[leftmargin=*]
  \item \textbf{Control plane (action):} ``may the agent call tool \emph{T} with parameters \emph{P}?'' ---
        the plane that pre-action authorization frameworks (OAP, PCAS, SEAgent, AgentGuardian) govern.
  \item \textbf{Data-result plane:} ``do the rows \emph{T} returns match the record-rule the user is entitled
        to?'' This is the axis we attack, in the setting where ERP governance is \textbf{incomplete}.
\end{itemize}

\subsection{Why ERP differs from warehouse-native}
Warehouse-native governed-NL-analytics (Snowflake Cortex Analyst, Databricks Genie, MS Fabric Data Agent)
\textbf{inherit complete governance} from the warehouse (RBAC, RLS, column masks applied uniformly at query
time). ERP (Odoo) enforces row security via \textbf{ORM record rules} that, in practice, \textbf{do not cover
every relation}: a team rule sits on the order header but not on its lines/payments/guarantees. Odoo record
rules are \textbf{default-allow} --- if an ACL grants access and no rule applies to the model/operation/user,
the rows are returned. An LLM agent enlarges the attack surface precisely because it autonomously chooses the
child/tool path a human rarely takes. (This direct-child-query bypass is documented native RLS behavior in
other engines too --- e.g.\ SQL Server applies a parent predicate only when the child is queried \emph{via}
the parent --- so the phenomenon is general; the novelty is its agent-driven exploitation in ERP + the
benchmark + the closure compiler.)

\subsection{Threat model (summary)}
Attacker = an employee probing beyond scope / prompt-injection / chaining, with a real role, natural-language
prompts only, no code or policy edits, observing refusal responses + latency. Defender = a deterministic PEP
at the data-result plane, with the \textbf{LLM kept outside the security boundary} (it does not decide
authorization --- OrgAccess shows GPT-4.1 reaches only F1$\approx$0.27 on RBAC reasoning). Out of scope:
infrastructure RCE, write/create/unlink, model extraction.

\section{ERP-AuthZBench}
A public, reproducible adversarial benchmark for row-level authorization of ERP LLM agents.
\begin{itemize}[leftmargin=*]
  \item \textbf{Schema mock} (\code{addons/pco_core_mock}): a 4-model sale cluster --- \code{pco.sale.order}

  \item \textbf{Two schema variants} (the heart of the benchmark): \textbf{V-vuln} = team rule on the header

  \item \textbf{Synthetic data} (\code{generate_synthetic.py}): deterministic (\code{seed=42}), generated ---
        never anonymized from production.
  \item \textbf{Attack suite} (\code{attacks.py}): 5 core classes (\code{relational-traversal},
        \code{aggregation-leak}, \code{sensitive-field-extraction}, \code{sensitive-measure-aggregation},

  \item \textbf{Oracle harness} (\code{evaluation_script.py}): each attack runs against a ground-truth oracle
        under three planes (native ir.rule, OAP-style action-authz, PG-Agent PEP); pass/fail is
        \textbf{measured, not asserted}. \emph{Caveat:} the headline numbers are oracle-based (deterministic
        ORM-level attacks); \S10.1.1 adds one \textbf{real-LLM run} over the public synthetic corpus as an
        empirical proxy, and the full end-to-end integration is validated separately in the private monorepo.
\end{itemize}

\section{PG-Agent: the data-result-plane PEP}
A model-agnostic PEP (\code{pg_agent_guard}) the agent must call instead of the ORM:
\begin{enumerate}[leftmargin=*]
  \item \textbf{Row-domain enforcement} --- inject a forced row-level domain (team/company/owner) per model,
        including the relation-traversal path for children (\code{order_id.team_code}); \textbf{fail-closed} on
        any model not in policy.
  \item \textbf{Sensitivity-aware masking} --- drop above-clearance fields \emph{before} the LLM context; deny
        confidential \code{read_group} measures and above-clearance group-keys.
  \item \textbf{Output validation} --- scan the final answer for leaked masked / cross-team values.
  \item \textbf{Uniform denial} --- identical empty result + constant-time/jitter, defeating
        existence-inference via the denial channel.
  \item \textbf{Independent audit} --- per-call decision log (stdlib, isolated from any AGPL audit module).
\end{enumerate}

(\code{policy_closure.py}, \code{domain_ast.py}, \code{policy_emit.py}) are offline-unit-tested; drivers run in

\subsubsection{Soundness frontier --- a proved pushdown theorem}
The \S5.3 gate is \textbf{sound but incomplete} (it refuses OR / multi-field outright). We prove a strictly
tighter sound gate (\code{pushdown_soundness.py}).

\noindent\textbf{Theorem.} Let $r$ be an F10 containment relation --- a stored Many2one that is \code{required}
(total) and \code{ondelete=cascade}, hence a \textbf{functional + total} map from each child $c$ to a unique
existing parent $\pi(c)$. For a parent domain $D_P$, the pushdown $D_P[f := r.f]$ admits exactly the children
$c$ whose $\pi(c)$ is admitted by the parent's \emph{operative} gate, provided (1) every leaf operator is a
stored \textbf{value-comparison} ($=,\neq,\mathtt{in},\mathtt{not\ in},<,>,\le,\ge,\mathtt{like},\ldots$) and
\textbf{not} hierarchical/subquery (\code{child_of}, \code{parent_of}, \code{any}, \code{not any}) nor
\code{=?}; and (2) \textbf{P-ACTIVE-CLEAN} --- if $P$ has an \code{active} field, the emit re-imposes
\code{(r.active,=,True)} (else manual-review). \emph{Proof (sketch):} $r$ functional+total $\Rightarrow$ each
value-comparison leaf $(r.f,op,v)$ on $c$ equals $(f,op,v)$ on $\pi(c)$; the connectives $\&/|/!$ are pointwise
on the same $c$ (the same $\pi(c)$); by structural induction $\mathrm{pushdown}(D_P,r)(c)=D_P(\pi(c))$.
\textbf{Boolean structure (AND/OR/NOT, multi-field) imposes no condition} --- it is always sound. $\square$

The operative parent gate is $D_P \wedge \mathrm{active\_test}(P)$: pushing only $D_P$ would admit a child of an
\textbf{archived} parent that the parent rule rejects --- so P-ACTIVE-CLEAN re-imposes \code{r.active}.
Hierarchical operators recurse over $P$'s own hierarchy; their dotted-rewrite semantics are
implementation-dependent, so we \textbf{withhold} them (manual-review), we do not \emph{refute} them. Reading
the verdict off \code{parse_domain}'s reason, the theorem lifts \textbf{1/5 $\to$ 3/5} soundly emittable on the
committed CE emit set (the \code{or/not} and \code{or/not;multi-field} gaps become sound; the two
\code{op:parent_of} gaps remain), and \textbf{1/6 $\to$ 4/6} on the Sales-app scan
(\code{results/scale/soundness.csv}) --- including \code{sale.order.template.line}, an \emph{active}-bearing
parent handled by re-imposing \code{sale_order_template_id.active} (the P-ACTIVE-CLEAN case). \textbf{Anti-claims:}
we do NOT soundly compile arbitrary domains (hierarchical/subquery/\code{active}-sensitive excluded);
\code{parent_of} is withheld, NOT refuted; the fraction is corpus-specific; the theorem refines the heuristic,
it does not obsolete it. The archived-parent counterexample is a regression test.

\subsection{ABAC/ReBAC formalization (RQ7)}
\code{results/policy_model.csv}. The bespoke per-model POLICY is, named explicitly, an \textbf{instance of a
general subject-context model}: each grant = a \textbf{ReBAC relation-path} (the M2O closure to the field's
defining model --- the same \code{_derive_path} BFS as \S5) $\times$ an \textbf{ABAC attribute-predicate}
(terminal field + operator + context RHS) $\times$ a \textbf{subject-context} of one of three kinds
\{group-membership, tenant-set, principal-id\}. \code{policy_model.py} (pure) \code{compile_policy} is a
faithful transcription of the guard's \code{_authz_domain}; a live round-trip proves \textbf{\code{compile_policy
== guard._authz_domain} for 20/20 persona $\times$ model} (Odoo 19). Every team/company \code{relation_path}
equals the PCC-ERP BFS closure (hops 0 on the header, 1 on a child); the company/owner contexts are recognized
ABAC tokens while \textbf{team is RBAC} (group-membership, resolved via \code{has_group} --- deliberately not a
domain context). \textbf{Honest: formalization, not new enforcement} --- it names what the guard already does
(zero new ir.rule/attack; \code{ci_gate} untouched), and ABAC over un-populated attributes (state/region ---
vacuous on the synthetic data) is deliberately omitted.

\section{Integrity --- RQ6 (applied / adopt-not-invent)}
A reliability threat orthogonal to authorization: an LLM agent that retrieves the \emph{right} rows can still
report a \emph{silently-wrong number} (a hallucinated or mis-derived value that looks plausible). We adopt the
warehouse-native principle --- \textbf{the LLM must not do arithmetic; every number binds to an execution
result} --- down to ERP. Framed explicitly as applied (not a research-novelty claim). No LLM in the public
artifact, so (as for authz) the demos are deterministic: planted answers + a trusted symbolic gold; what is
demonstrated is the \emph{mechanism}, with the real-LLM rate validated privately. Three complementary layers
(TB.1/TB.2/TB.3), a strict division of labor:

\subsection{Numeric verifier (TB.1)}
\code{results/integrity.csv}. A pure, offline-tested scanner: each answer number must bind to the governed
execution table or a \textbf{bounded derivation} of it (sum / pairwise-diff / ratio\% / pct-change /
share-of-total; magnitude + rounding tolerance; VN/EN decimals). 6 questions across \textbf{5 kinds}
(aggregation / ratio / growth-\% / period-comparison / multi-step), each with a symbolic gold.

\begin{center}\small
\begin{tabular}{@{}ll@{}}
\toprule
metric & result \\
\midrule
Silently-Wrong-Number Rate & raw text-to-ORM \textbf{6/6} (wrong present) $\to$ \textbf{+verifier 0/6} \\
false-flag rate on correct/derived answers & \textbf{0/6} (passes growth-\%, ratios, negative diffs) \\
coverage & \textbf{5/5 kinds} \\
\bottomrule
\end{tabular}
\end{center}

TB.1 catches numbers \textbf{not derivable} from the data (fabricated / cross-data). Its \textbf{blind spot}:
\emph{correct-arithmetic-with-the-wrong-formula} --- a number that \emph{is} a valid derivation (so it binds)
yet answers a \emph{different} question (wrong rows/field/agg/dimension).

\midrule

\section{Doc-RAG retrieval plane (RQ8)}
The PEP extends from the structured-data plane to a \textbf{retrieval} plane. A RAG agent retrieves document
\textbf{chunks} (derived from records) to answer a question; the confused-deputy is the \textbf{retriever},
which ranks the most \emph{relevant} chunks regardless of whether the persona may read the source record. The
mechanism reuses the data-plane guard: \code{guard.guarded_retrieve} routes each retrieved chunk's provenance
back through \code{guarded_search_read} --- a chunk whose source record is \textbf{not row-authorized is
dropped}, and a survivor is delivered only \textbf{re-rendered from the clearance-masked source}. A
deterministic lexical (term-overlap) retriever stands in for the embedding ranker; the security property is
\textbf{independent of the ranker}. No LLM.

Oracles are \textbf{independent} of the guard's verdict: the full row-authz permitted set (unauthorized-row
delivery) and the SUDO cleartext value (\code{output_validator} as a presence scanner, not the guard's own
verdict). \code{results/docrag.csv}:

\begin{center}\small
\begin{tabular}{@{}lcc@{}}
\toprule
attack & undefended (unauth / confid) & guarded \\
\midrule
cross-team-direct (``nh\'om ttf'' query)               & 8/8  & \textbf{0/0} \\
cross-team-incidental (generic query, spans teams)     & 8/12 & \textbf{0/0} \\
confidential (own-team, spans masked)                  & 4/8  & \textbf{0/0} \\
utility / false-block                                  & 4/8  & \textbf{0/0} \\
\bottomrule
\end{tabular}
\end{center}

\textbf{Undefended leaks 60 $\to$ guarded unauthorized 0, confidential 0, false-block 0} (Odoo 19);
\code{ci_gate} unaffected (the driver self-asserts, like \S5.4, rather than gating). \textbf{Honest scope:} the
PEP gates \textbf{delivery, not the index} --- cleartext indexing is an assumption and rank order can be
influenced by confidential content (a named residual, not a value leak). Structural masking avoids the
output-validator paraphrase residual \emph{because} chunks are provenance-tracked; true unstructured prose has
no provenance, falls back to content-scanning, and inherits that residual (carried as one out-of-scope
free-prose probe). \textbf{No real-LLM / embedding RAG rate is claimed.}

\section{Related work \& positioning}
\begin{itemize}[leftmargin=*]
  \item \textbf{Governed + secure NL analytics (prior art for the ``unified'' pillar):} Cortex Analyst,
        Databricks Genie, MS Fabric Data Agent (industry); SAFEFLOW (IFC, integrity+confidentiality). All
        \textbf{assume complete/inherited governance} and do not target ERP record rules or adversarial
        row-level testing.
  \item \textbf{Action/flow-plane authorization for agents:} OAP, PCAS, SEAgent, AgentGuardian, ClawGuard,
        AgentBound --- govern the call, \textbf{not} the result rows; we use an OAP-style baseline.
  \item \textbf{Database access-control synthesis:} DePLOI/IBAC-DB synthesizes/audits \textbf{table-level}
        grants from NL intent; OPA partial-eval / predicate pushdown compile policy$\to$filter. PCC-ERP differs
        in \textbf{relational-closure derivation across the ORM graph} (which child needs which parent's
        pushed-down rule), not the filter mechanics.
  \item \textbf{RBAC / agent-security benchmarks:} OrgAccess (RBAC reasoning, GPT-4.1 F1$\approx$0.27 ---
        motivates LLM-outside), ASB, SecureMCP (table/column RBAC), Role-Conditioned Refusals (text-to-SQL).
  \item \textbf{Concurrent agent-authorization work (2025--26)} --- a fast-moving wave, all \emph{orthogonal
        to the data-result plane}: $\tau$-bench and PolicyBank evaluate/refine agent adherence to
        natural-language \textbf{policy text} at the tool-capability level (not row-level DB enforcement);
        mandatory-access-control frameworks for multi-agent \textbf{privilege escalation} (the ``Taming
        Privilege Escalation''/SEAgent line, 0\% ASR across prompt-injection / RAG-poisoning / confused-deputy
        \emph{across agents}) and Agent-GrantBox (least-privilege at the \textbf{tool/API} interface) govern
        the call/permission, not the returned rows; ``A Vision for Access Control in LLM Agent Systems'' is a
        position paper. None targets ERP record-rule incompleteness on child models or the relational-traversal
        row bypass.
  \item \textbf{Denial-channel leakage:} ``Causality Laundering: Denial-Feedback Leakage in Tool-Calling LLM
        Agents'' \textbf{formalizes} (causal theory) and \textbf{detects} information leakage through an
        agent's refusal/denial feedback --- the same channel our \textbf{uniform-denial} layer (T2.4/RQ5)
        attacks, but from the opposite end: they characterize/measure the leak, we \textbf{operationally
        close} the observable (identical empty result + constant-time, measured as Existence-Inference Rate
        1/1 $\to$ 0/1, \S4.4). Complementary: their lens is the theory of the channel; ours is a deployable PEP
        defense + a benchmark metric for it.
\end{itemize}

\textbf{Novelty (honest):} the \emph{combination} --- ERP record-rule incompleteness on child models $\times$
agent-driven relational-traversal exploitation $\times$ the first adversarial benchmark $\times$ a
relational-closure compiler --- not any single mechanism, and (per the 2025--26 wave above) \textbf{the
data-result plane is the un-occupied niche}: concurrent agent-authorization work targets the
control/permission plane, policy-text compliance, multi-agent MAC, or the denial-channel \emph{theory}; none
enforces or compiles \textbf{row-level} ERP record rules. We explicitly do \textbf{not} claim a novel
authz+integrity unification (prior art) or a general policy compiler for agents (PCAS/OAP-adjacent). Subtitle
framing: \emph{``Policy-Closure Compilation for Row-Level Authorization in ERP LLM Agents.''}

\section{Limitations \& honest scope}
\begin{itemize}[leftmargin=*]
  \item \textbf{Oracle harness, real-LLM run is one proxy (public artifact):} the headline attacks are
        deterministic ORM-level probes; one \textbf{real-LLM run} (\S10.1.1, \code{gpt-4o-mini}, N=12, synthetic)
        gives an empirical proxy (ASR-without-guard 2/12, guarded 0/12) --- not a stable rate; the full
        production loop is validated privately. The PEP correctness claim is at the data-result plane.
        The integrity layers (\S6) likewise demonstrate \emph{mechanisms} on planted answers/candidates --- not
        measured LLM hallucination or wrong-formula rates --- and the governed-metric coverage is partial by
        design (hybrid).
  \item \textbf{CE gap count is modest:} standard Odoo is broadly company-governed; the 5 gaps are real but the
        heavier gap/burden target is the bespoke \code{pco_core} or larger module sets.
  \item \textbf{Emit soundness is bounded but characterized (\S5.3.1):} the conservative gate emits only single
        simple leaves (1/5 on CE); a \textbf{proved theorem} widens this to any boolean structure of stored
        value-comparison leaves through a required-cascade M2o (1/5 $\to$ 3/5 on CE, 4/6 on the Sales-app scan),
        with \code{active}-bearing parents handled by re-imposing \code{r.active}. The remaining frontier
        (hierarchical \code{parent_of}/\code{child_of}, subquery) is withheld (manual-review), not refuted;
        sound pushdown of \emph{arbitrary} domains is not claimed.
  \item \textbf{Owner axis} is a local opt-in field, out of relational-closure scope by construction.
  \item \textbf{Red-team is grammar-exhaustive, not exhaustive (\S4.3):} the automated red-team enumerates a
        \emph{defined} ORM-pivot grammar (a modeled threat surface) deterministically and without an LLM; a
        green gate means no bypass at any enumerated point, not a universal-correctness proof.
  \item \textbf{RQ7 is formalization, not enforcement (\S5.4):} it names the team/company/owner predicates the
        guard already enforces (no new ir.rule/attack); ABAC over un-populated attributes would be vacuous and
        is omitted.
  \item \textbf{RQ8 gates delivery, not the index (\S7):} the retrieval PEP re-checks provenance at delivery;
        the cleartext index and rank-order are named residuals, structural masking depends on field provenance
        (unstructured prose inherits the paraphrase residual), and no real-LLM / embedding RAG rate is claimed.
  \item \textbf{Read-scoped:} write/create/unlink, prompt-injection elimination, and infrastructure threats are
        out of scope; prompt injection is ``reduced + measured'', not ``eliminated''.
\end{itemize}

\subsection{Private validation (corroborating, not reproducible)}
Because the guard rewrites the query domain \textbf{below the LLM trust boundary} --- the model only emits ORM
tool-calls and the PEP forces the authorization domain before the database --- the row-level security property
is \textbf{independent of which LLM (or whether any LLM) drives the loop}; the deterministic public proof
(leak 0 on every class, variant, and enumerated grammar point) is the security proof, and the private real-LLM
run measures only what the mechanism does not guarantee a priori (utility, the answer-channel paraphrase
residual, and wrong-formula rate).

The private end-to-end rates are obtained with the \textbf{same instrument} as this artifact ---
\code{evaluation_script.py} / \code{ci_gate}, the public \code{PERSONAS}, and the public ERP-AuthZBench +
adaptive + red-team attack sets --- pointed at the production data with a real tool-calling model via the
\code{LLMAgent} seam (\S10.1). \textbf{Only the data and the resulting numbers are private; the measurement
instrument is the audited public one.} A reader reproduces the \emph{method} by instantiating \code{LLMAgent}
against their own model + the synthetic \code{seed=42} corpus.

Honest scope of this section: \textbf{(1)} the private numbers carry \textbf{no headline security claim} --- the
deterministic proof (\S4) and the reproducible proxy (\S10.1) carry the security load; the private run only
\emph{corroborates} and characterizes utility/residual. \textbf{(2)} They are \textbf{privately measured, not
reproducible from this artifact} (real data + a real model the public repo deliberately does not contain).
\textbf{(3)} The committed proxy (\S10.1) is a \emph{mechanism + utility} demo, \textbf{not} a stand-in for the
real-LLM rate. \textbf{(4)} We \textbf{report negative results}: any case the private real-LLM loop surfaces
that the deterministic harness missed is fed back as a new public ERP-AuthZBench variant --- that feedback
loop, not a clean private number, is the evidence of good faith. \emph{(Production figures and an
internal-attestation line are filled in the private build; they are omitted here rather than shown as
placeholders.)}

\section{Conclusion}
ERP LLM agents leak rows at the data-result plane when record-rule governance is incomplete on child models ---
a gap the warehouse-native and action-plane lines of work do not close for ERP. ERP-AuthZBench measures it,
PG-Agent closes it (clean on every class, in both schema variants, robust to adaptive path-switching), and
PCC-ERP shows the per-model closures are \emph{derivable}: it reconstructs the guard policy on the mock
end-to-end and surfaces 5 genuine relational-traversal gaps in vanilla Odoo CE, emitting sound native rules
exactly where the parent rule is pushdownable. On the reliability axis (RQ6), the three-layer integrity stack
drives the silently-wrong-number rate to zero and closes the numeric verifier's wrong-formula blind spot (TB.1
0/6 $\to$ +TB.3 4/6 $\to$ +TB.2 6/6) via deterministic governed metrics + execution-voting. Beyond the
canonical results, the residual-risk surface is regression-gated by an exhaustive ORM-pivot \textbf{red-team
grammar} (residual-leak 0/41); the bespoke POLICY is shown to be an instance of a general
\textbf{ABAC$\times$ReBAC} subject-context model whose formal compiler reproduces the guard's exact domain
(RQ7, 20/20); and the PEP extends to a \textbf{Doc-RAG retrieval plane} that delivers chunks only re-rendered
from row-authorized, clearance-masked sources (RQ8, guarded leak 0). The work is applied-security + benchmark +
reference implementation for an under-served setting, with an explicitly honest soundness frontier.
